스프링 입문 - 코드로 배우는 스프링 부트, 웹 MVC, DB 접근 기술의 일부를 듣고 정리하는 글 입니다.

JAVA 설치

Mac에서 HomeBrew로 그냥 최신 버전 설치 했다.

IDE 설치

이전에 잠깐 회사에서 스프링으로 프로젝트를 할 때는 이클립스를 썼었는데

이번에는 인텔리제이를 찍먹 해보고 싶어서 커뮤니티 버전으로 다운 받아서 설치를 했다.

프로젝트 구성

Spring initializr에서 강의 내용 기반으로 설정해주었다.

강의와 다른 점은 현재 스프링부트 안정 버전은 2.4.2, 자바 버전은 내 컴이 15이므로 15로 설정 하였다.

프로젝트 실행

인텔리제이에서 Open -> 소스의 build.gradle -> Open as Project를 한다.

그러면 필요한 패키지 등을 다운을 알아서 받게 되는데 약간의 시간 소요가 된다.

이후 java 폴더에서 MyAppApplication 파일에서 main 메소드를 실행해주면 된다.

View 파일 설정

웰컴 페이지

루트로 접속 했을 때 나오는 정적 페이지를 웰컴 페이지라고 부르는 거 같았다.

resources -> static -> index.html을 생성해서 적당히 내용을 작성 하고

톰캣 재실행 후 접속을 하게 되면 해당 페이지로 뜬다.

컨트롤러 및 뷰 템플릿 설정

컨트롤러 패키지를 생성 후 유저 컨트롤러라는 클래스를 만들어서 작성을 하였다.

@ 붙은(어노테이션이라 부르는 것)은 시작은 대문자이며, 작성 중에 IDE에서 박스가 뜨면서 선택 할 수 있다.

암튼 단어가 생각 안 나는데.. 😰 선택하면 관련된 라이브러리를 자동으로 import 해 준다. (이건 이클립스에서도 동일)

이전에 스프링 할 때는 프리마커를 조금 써 봤었는데 (문법 기억도 안남) 타임리프는 처음 들어 봤다.

resources -> templates 내에 작성을 하게 되며 파일 확장자는 html 그대로 유지를 해도 된다.

배운 점 & 느낀 점

  • 자바랑은 그다지 친하지 않아서.. 이 참에 코틀린을 건드려볼까? 했는데 일단 언어는 자바로 🧐
  • 이전에 스프링 프로젝트를 했을 때는 이클립스에서 설정할 게 꽤 많고 불편 및 귀찮아서 거부감이 좀 드는 편이였는데 부트는 간소화?가 되어서 마음에 든다.
  • 요즘은 메이븐 보단 그레들로 외부 라이브러리를 관리하는 게 추세라고 한다.
  • 스프링부트 스타터에는 스프링 관련 및 자주 쓰는 외부 라이브러리가 내장 되어 있는 듯 하다. (톰캣, 로깅 관련, json 관련 잭슨 등등..)
  • 이전에는 JSON 응답하려면 클래스에서 외부 라이브러리로 뭔가 변환해주는 작업을 했었던 거 같은데.. 지금은 안 해도 되는걸로 😮
