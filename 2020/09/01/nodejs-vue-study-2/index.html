컴포넌트 분할

컴포넌트 구조를 잘 나눠줘야 유지 하기가 편해지고 재사용 하기 좋다고 해서 고민을 조금 했었다.

큰 단락으로 구분을 나누어서 진행을 할 지, 작은 단위로 쪼개야 할지..

작게 역할별로 쪼개면 지금은 아니지만 실무에서는 다른 사람이 봐도 조금 더 편하지 않을까란

생각이 들어서 일단은 작은 단락으로 쪼개서 해보기로 했다.

메뉴

메뉴의 경우 메인 메뉴와 서브 메뉴가 있으며 서브 메뉴가 있는 경우에는 toggle 버튼 방식으로

클릭 시 애니메이션이 나오면서 등장해야 하고 서브 메뉴가 없는 경우에는 페이지 이동을 하는

구조 여서 메인 메뉴는 v-for 지시자를 이용해 만들었고, 서브 메뉴의 경우는 해당 메뉴 일 때

v-show 지시자를 이용해 만들었으나 문제는 active 클래스 추가나 서브 메뉴 애니메이션이..

기존 jQuery를 이용한 코드

:class 지시자를 통해 active 클래스는 제어가 되었으나 슬라이드 애니메이션에서 막혀 버렸다.

트랜지션을 이용해서 이리 저리 해봤지만 애니메이션이 매끄럽지 않아서 일단 포기 했다.

애니메이션은 제이쿼리나 기타 라이브러리에 의존을 했었기에 지식 부족 인게 느껴졌었다 😭
