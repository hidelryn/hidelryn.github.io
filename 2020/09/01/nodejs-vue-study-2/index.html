컴포넌트 분할

컴포넌트 구조를 잘 나눠줘야 유지 하기가 편해지고 재사용 하기 좋다고 해서 고민을 조금 했었다.

큰 단락으로 구분을 나누어서 진행을 할 지, 작은 단위로 쪼개야 할지..

작게 역할별로 쪼개면 지금은 아니지만 실무에서는 다른 사람이 봐도 조금 더 편하지 않을까란

예약어인지 금지어인지 모르겠으나 조심을 해야 겠다.

이미지 삽입

메뉴

v-for 지시자를 이용해 메뉴들을 반복문으로 처리를 하는 중 v-for에 대한 key를 별도로

적어주지 않으면 lint에 걸리길래 심플하게 index로 처리하였다.

하위 메뉴가 없는 경우 v-if를 통해 제어를 하려고 v-for와 v-if를 같은 라인에서

적용 헤봤으나 lint에서 쓰지 마시오 라는 경고가 떠서 수정을 했었다.

메뉴 클릭 시 active 효과와 토글 버튼 슬라이드 효과 등은 아직 Vue로 바꾸기가 힘들어서

다음 번 업데이트 때 수정을 하기로 하고 😭 지금은 기존 제이쿼리를 사용하기로 했다.

다행히 event.target을 지원을 하여서 제이쿼리 쪽은 큰 문제 없었다.

router-link

template 속성에 a 태그를 그대로 사용을 하였더니 메뉴를 누르게 되면 화면이 깜빡이면서

전환이 되길래 다시 책을 살펴 보고 router-link로 교체를 바꿨더니 문제가 없어졌다
